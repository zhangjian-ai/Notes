# 海外手游出海全流程指南
## 海外手游测试体系
### 海外游戏测试 vs 传统APP测试 核心区别
| 对比维度     | 海外手游测试                  | 传统APP测试              | 海外特殊重点                 |
| :----------- | :---------------------------- | :----------------------- | :--------------------------- |
| **核心目标** | 可玩性、稳定性、合规、本地化  | 业务流程、功能稳定       | 文化适配、全球网络、分级合规 |
| **功能测试** | 游戏逻辑+数值+社交+联机       | 页面交互、接口、流程闭环 | 抽卡概率、多语言、跨服同步   |
| **性能测试** | FPS、发热、掉电、资源加载     | 启动、内存、响应         | 低端机适配、弱网、跨境延迟   |
| **网络测试** | 全球节点、弱网/丢包、重连     | 国内网络、接口容错       | Socket 实时同步、跨境专线    |
| **合规测试** | GDPR/COPPA、ESRB/PEGI 分级    | 国内隐私、权限           | 数据跨境、儿童隐私、内容审查 |
| **SDK 测试** | 海外支付、广告、Facebook/Line | 国内支付、推送、统计     | 区域渠道SDK、支付回调        |
| **安全测试** | 反作弊、防破解、存档加密      | 账号安全、接口安全       | 外挂检测、服务器校验         |

### 海外游戏必做测试项
1. **本地化测试**
   - 多语言文字、UI 适配、时间/货币/时区格式
   - 文化禁忌、区域节日、美术素材合规
2. **合规测试**
   - GDPR/CCPA 隐私政策、用户数据删除权
   - 地区分级（北美 ESRB / 欧洲 PEGI / 日本 CERO）
3. **网络&稳定性测试**
   - 全球区域网络（欧美/东南亚/拉美）弱网模拟
   - 断网重连、切换网络、高延迟、高丢包场景
4. **性能&兼容性测试**
   - 帧率、发热、内存、包体、资源加载
   - 海外主流中低端机型覆盖
5. **支付&SDK测试**
   - Google Pay / Apple Pay / 区域本地钱包
   - 支付成功/失败/取消/退款/掉单全流程
6. **反作弊测试**
   - 内存修改、加速器、脚本、封包篡改
   - 关键逻辑必须**服务器校验**

### 常用测试工具
- 弱网：Charles、Fiddler、Qnet
- 崩溃/日志：Firebase Crashlytics、Bugly
- 海外真机：BrowserStack、Testin
- 自动化：Appium、Hypium、Airtest



## 海外游戏发布上架流程

### 上架前置必备准备
| 项 目      | 说明                                         | 海外关键点                       |
| :--------- | :------------------------------------------- | :------------------------------- |
| 开发者账号 | Google Play（$25/终身）、App Store（$99/年） | 企业资质更稳定，个人号有测试限制 |
| 合规文件   | 软著、隐私政策（HTTPS）、分级证明            | 多语言隐私政策、数据跨境合规     |
| 安装包     | Android：AAB；iOS：IPA                       | Google 强制 Play App Signing     |
| 素材       | ICON、截图、视频、描述、关键词               | 区域ASO、美术符合当地审美        |

### Google Play 上架流程
1. 上传 AAB → 开启**封闭测试**（≥100 人，≥14 天，个人号强制）
2. 开放测试扩量 → 收集 Crash、反馈、优化
3. 填写商品信息、分级、隐私、支付配置
4. 提交审核 → 1~3 天 → 发布上线/分区域发布

### App Store 上架流程
1. 上传 IPA → TestFlight 内/外部测试
2. 完善信息、审核备注、权限说明
3. 提交审核 → 3~7 天
4. 审核通过 → 自动/手动发布
5. 紧急 Bug 可申请**加急审核**

### 海外发行特殊环节
- 部分地区单独审查：台、韩、日等
- 区域渠道：Garena（东南亚）、OneStore（韩国）
- 灰度策略：新马小服先跑 → 稳定后推欧美日大服



## 用户反馈 & 收益 实时监测

### 用户反馈实时监测
#### 反馈渠道与工具
| 渠道     | 监测方式                      | 推荐工具             |
| :------- | :---------------------------- | :------------------- |
| 应用商店 | 评分、评论、差评聚类          | Appbot、Sensor Tower |
| 游戏内   | 崩溃、日志、反馈工单          | Firebase、Helpshift  |
| 社媒     | Discord、Reddit、Twitter 舆情 | Affogata             |
| 客服     | 工单自动分类、告警            | Zendesk              |

#### 实时技术架构（可落地）
- 采集：客户端埋点/Socket 上报 → Kafka
- 计算：Flink 实时计算、异常检测
- 存储：ClickHouse 实时数仓
- 展示：Grafana 大盘 + 告警
- 部署：K8s 集群

### 收益实时监测
#### 核心实时指标
- 支付：转化率（用户付费率）、ARPU（每用户平均收益）、ARPPU（每付费用户平均收益）、LTV（每个用户的总收益）、退款率
- 广告：eCPM、填充率、广告总收益
- 渠道：下载、激活、ROI（投入产出比）

#### 监测工具
- 通用：GameAnalytics、Adjust、AppsFlyer
- 支付订阅：RevenueCat
- 自研：Java 后端对接渠道回调 + 实时大屏





## 可直接落地 · 出海全流程Checklist

### 测试阶段Checklist
- [ ] 多语言文字、UI 无截断、无乱码
- [ ] 隐私政策多语言、合规可访问
- [ ] 抽卡/概率公示符合当地法规
- [ ] 全球弱网/高延迟/丢包测试通过
- [ ] 支付全流程（成功/失败/退款/掉单）通过
- [ ] 崩溃率 < 0.5%，无致命Crash
- [ ] 反作弊关键逻辑走服务器校验
- [ ] 海外主流中低端机帧率、发热达标

### 上架发布Checklist
- [ ] 开发者账号正常、无封禁风险
- [ ] 隐私政策 HTTPS 地址可用
- [ ] 分级信息与内容一致
- [ ] 包名、签名、配置统一无误
- [ ] 商店素材、描述、关键词完成本地化
- [ ] 测试充分，无高频崩溃
- [ ] 灰度区域、发布节奏已规划
- [ ] 支付、广告、统计SDK已配置

### 监测&告警Checklist（建议阈值）
- [ ] 崩溃率 ≥ 1% → 实时告警
- [ ] 单区域评分 ≤ 3.5 → 告警
- [ ] 支付失败率 ≥ 5% → 告警
- [ ] 差评突增/“登不上”“扣钱”关键词 → 告警
- [ ] 收益异常波动 ≥ 20% → 告警
- [ ] 广告 eCPM（每一千次展示可以获得的广告收入） 突降 ≥ 30% → 告警
- [ ] 核心接口延迟突增 → 告警

### 应急处理SOP
- 严重 Crash/无法登录：1h 定位 → 24h 热更/补丁
- 支付异常：立即切备用渠道，同步对账
- 差评爆发：快速公告+补偿+版本修复
- 审核被拒：按平台意见修改，加急重提



## 极简团队执行总结

1. **测试**：本地化、合规、网络、性能（帧率、发热、耗电）、支付&广告&统计SDK、崩溃&Error&ANR、防作弊&防破解
2. **上架**：先测试、再灰度、后全量，重视审核规范
3. **监测**：实时看崩溃、评分、支付、收益，告警即处理
4. **核心**：海外 = 合规 + 网络 + 本地化 + 实时闭环
