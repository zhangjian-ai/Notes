### 测试阶段划分

> 个性化推荐模型的测试是一个系统性工程，需结合离线评估、在线实验和长期监控三个阶段进行。

- 离线测试（Offline Testing） 
  在历史数据集上验证模型基本有效性，快速筛选候选模型。  

- 在线测试（Online Testing / A/B Testing）

  将模型部署到真实流量中，通过用户行为评估实际效果。  

- 长期监控（Long-term Monitoring） 

  模型全量上线后持续跟踪性能，防止衰减或偏差。



### 核心评估指标

> 推荐系统应根据业务场景权衡指标权重：电商侧重CTR与转化率，内容平台更关注多样性与新颖性 。

| 类别     | 指标                                                         | 说明                                                         |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 准确性   | 准确率（Precision TP/(TP+FP)）、召回率（Recall TP/(TP+FN)）、F1分数（精确率与召回率的调和平均） | 衡量推荐结果与用户真实兴趣的一致性                           |
| 排序质量 | NDCG（Normalized Discounted Cumulative Gain 归一化折损累计增益） | 衡量推荐列表是否将高相关性内容排在前面。NDCG@10 > 0.7 通常视为优秀，广泛用于搜索与推荐系统排序优化。 |
| 多样性   | 覆盖率（Coverage）、新颖性（Novelty）                        | 避免推荐同质化，提升长尾内容曝光                             |
| 商业价值 | CTR（点击率）、CVR（转化率）、GMV（商品交易总额）、留存率    | 直接关联业务目标                                             |
| 公平性   | 子群体偏差分析                                               | 检查模型对不同用户群体（如新/老用户）是否存在歧视            |



### 主流测试方法

1. A/B 测试  
   - 将用户随机分组为实验组（使用新模型）和对照组（使用基线模型）。  
   - 控制变量（如流量比例、实验周期≥1周），对比核心指标差异 。  
   - 示例：实验组CTR提升10%且统计显著，则模型有效 。

2. 离线交叉验证  
   - 使用历史行为数据（如MovieLens），按时间划分训练集/测试集，避免数据泄露。  
   - 常用K折交叉验证，评估RMSE（均方根误差）、NDCG等指标 。

3. 用户反馈与主观评估  
   - 通过问卷、NPS评分或“不感兴趣”按钮收集主观满意度 。  
   - 结合点击、收藏、停留时长等行为数据量化体验 。

4. 公平性与偏差专项检测  
   - 分析模型在不同用户子群体（如性别、地域、活跃度）上的表现差异。  
   - 防止“马太效应”（高活跃用户持续受益，低活跃用户被忽视）。



### 实施建议

> 流程闭环：离线筛选 → A/B验证 → 全量上线 → 持续监控 。  

#### 离线测试：基于历史数据的初步验证

在模型上线前，利用历史用户行为数据（如点击、购买、评分）进行模拟评估。

- ‌**数据划分**‌：将数据按时间或随机划分为训练集（80%）和测试集（20%），避免数据泄露。
- **核心指标计算‌**：
  - ‌**准确率（Precision）**‌：推荐列表中用户真正感兴趣的内容占比。
  - ‌**召回率（Recall）**‌：用户感兴趣的内容中被成功推荐的比例。
  - ‌**F1分数**‌：准确率与召回率的调和平均，衡量整体推荐质量。
  - ‌**NDCG@K**‌：考虑排序位置的评估指标，越靠前命中权重越高，适用于Top-K推荐场景。
- ‌**工具支持**‌：使用Python的scikit-learn、TensorFlow等库自动化计算指标。

> 示例：在MovieLens数据集上测试协同过滤模型，若NDCG达到0.65以上，说明排序质量较好。



#### 在线测试：真实环境下的效果验证（A/B测试）

将新模型部署至部分真实用户流量中，对比其与旧模型的表现差异。

- **实验设计**：
  - 用户随机分为实验组（新模型）与对照组（原系统），确保两组用户分布一致。
  - 实验周期建议2–4周，覆盖不同用户活跃周期。
- ‌**关键评估指标**：
  - ‌**点击率（CTR）**‌：反映推荐内容吸引力。
  - ‌**转化率（CVR）**‌：衡量推荐对交易行为的影响。
  - ‌**停留时长、页面浏览量（PV）**‌：体现内容匹配度。
  - ‌**用户留存率**‌：长期粘性指标，判断是否提升用户体验。
- ‌**结果判定**‌：通过统计显著性检验（如p值 < 0.05）判断提升是否可靠。

> 案例：某商城优化“猜你喜欢”功能后，A/B测试显示实验组CTR提升12%，且转化率显著上升，模型通过验证。



#### 长期监控与持续优化

模型全量上线后需持续追踪其表现，防止性能衰减或出现偏差。

- **‌监控维度**：
  - ‌**算法性能**‌：每日跟踪准确率、召回率变化趋势。
  - ‌**用户体验**‌：收集“不感兴趣”反馈、用户投诉等主观信号。
  - ‌**公平性**‌：分析不同用户群体（如新/老用户、地域差异）的推荐覆盖率，避免歧视。
  - ‌**多样性与新颖性**‌：确保推荐不局限于热门内容，提升长尾物品曝光。
- ‌**反馈机制**：
  - 建立实时日志系统，结合用户行为动态调整模型。
  - 定期重新训练模型，适应兴趣漂移。



### 补充测试方法

- ‌**用户满意度调查**‌：通过问卷或评分系统直接获取主观反馈。
- ‌**多模型对比测试**‌：同时运行多个推荐策略，选择最优方案。
- ‌**冷启动专项测试**‌：针对新用户或新物品设计独立评估流程，确保推荐可用性。
  - 冷启动非常关注AUC（模型给「点了的内容」打分 > 「没点的内容」打分的概率）指标，AUC不受样本数量不平衡问题的影响（推荐场景天然存在点击、不点击样本的不对等）
  - AUC越接近 1，模型越能把用户想点的东西排前面；越接近 0.5，越等于乱排
